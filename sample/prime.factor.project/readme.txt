■  prime factor project(solomon sample project)
■


■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
■
■  コンパイル環境前提条件
■

・Visual C++ がインストールされていること。
　→ 2012 および 2013 で正常にコンパイルができることを確認しています。
　→ Visual C++ のバージョンやインストール先ディレクトリなどによっては
　　適宜 .\src\solomon\build ディレクトリ内(サブディレクトリ内も含む)の
　　*.cmd を修正する必要があります。


■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
■
■  構成
■

■.\src ディレクトリ
　→スナップショットが作成される対象となるディレクトリであり、ソースコード
　　やビルドバッチ一式が全て格納されるディレクトリです。

■.\src\prime.factor.cpp
　→このサンプルプロジェクトのソースコードです。
　→引数で与えられた整数値の素因数分解を行います。
　→自己テストコードが同梱されており AUTO_SELF_TEST_BUILD マクロがdefineされ
　　ている状態でコンパイルされると自己テストプログラムとなります。

■.\src\solomon ディレクトリ
　→ ..\..\cmd\template\solomon をベースにこのサンプルプロジェクト固有の
　　設定とビルドバッチを加えたものになります。

■.\src\solomon\build.cmd
　★このバッチを実行するとビルドが実行されその結果に応じてスナップショット
　　が作成されます。

■.\src\solomon\auto.build.cmd
　★このバッチを実行するとビルドが実行されその結果に応じてスナップショット
　　が作成されます。
　→ build.cmd と異なり、ビルド終了後は待機状態となり .\src ディレクトリ内
　　(サブディレクトリ内も含む)のファイルが更新される度にビルドが実行されます。

■.\src\solomon\conf\config.cmd
　→このサンプルプロジェクト固有の solomon の設定。
　★SOLOMON_SNAPSHOT_HISTORY を TRUE に指定するとスナップショットが上書
　　きではなく履歴として残るようになるので是非お試しください。
　★7zip がインストールされているならばさらに SOLOMON_SNAPSHOT_HISTORY_DIR_FORMAT 
　　を \YYYY\MM\DD\hhmmss.zip に変更することで履歴が ZIP ファイルとして
　　作成されますので、こちらも是非お試しください。

■.\snapshot\master ディレクトリ
　→ビルドが実行される際にまず最初に .\src ディレクトリのコピーがここに作成
　　されます。
　→初期状態では存在しません。

■.\snapshot\missed.compile ディレクトリ
　→最後にコンパイルエラーが発生した時のスナップショットがここに保存されます。
　→設定によっては最後のスナップショットだけでなく履歴として保存されます。
　→初期状態では存在しません。

■.\snapshot\passed.compile\missed.test ディレクトリ
　→最後にコンパイルが成功し且つテストでエラーが発生した時のスナップショット
　　がここに保存されます。
　→設定によっては最後のスナップショットだけでなく履歴として保存されます。
　→初期状態では存在しません。

■.\snapshot\passed.compile\passed.test ディレクトリ
　→最後にコンパイルとエラーが成功した時のスナップショットがここに保存されます。
　→設定によっては最後のスナップショットだけでなく履歴として保存されます。
　→初期状態では存在しません。

■.\snapshot\result\*\prime.factor.exe
　→目的のコマンドライン実行形式ファイル。
　→引数で与えられた整数値の素因数分解を行います。
　→初期状態では存在しません。

■.\snapshot\result\*\prime.factor.test.exe
　→自己テストを行うコマンドライン実行形式ファイル。
　→ AUTO_SELF_TEST_BUILD マクロがdefineされた状態でコンパイルされた結果。
　→初期状態では存在しません。

■.\snapshot\result\*\test.result.html
　→自己テストプログラムにより行われた自己テストの結果。
　→初期状態では存在しません。

■.\snapshot\solomon.data ディレクトリ
　→solomon がデータを格納するディレクトリ。
　→.\src\solomon\build.cmd や .\src\solomon\auto.build.cmd の実行中でな
　　ければ削除しても構いません。
　→初期状態では存在しません。

■.\testshot ディレクトリ
　→実際にビルドが実行されるワークディレクトリ。
　→ビルドが行われる度に .\snapshot\master と同一の内容に初期化された上で
　　ビルドが実行されます。
　→初期状態では存在しません。
